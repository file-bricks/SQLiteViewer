# Security Policy / Sicherheitsrichtlinie

## Sicherheitsluecken melden

Bitte melden Sie Sicherheitsluecken verantwortungsvoll:

1. **Kein oeffentliches Issue erstellen**
2. **GitHub Private Vulnerability Reporting verwenden**:
   https://github.com/file-bricks/SQLiteViewer/security/advisories/new
3. Beschreibung, Reproduktionsschritte und moegliche Auswirkungen angeben

SQLiteViewer verarbeitet lokale Datenbankdateien. Relevante Meldungen betreffen
insbesondere Dateizugriff, SQL-Ausfuehrung und Exportfunktionen.

## Reporting a Vulnerability

If you find a security vulnerability, please report it responsibly:

1. **Do NOT open a public issue**
2. **Use GitHub's [private vulnerability reporting](https://github.com/file-bricks/SQLiteViewer/security/advisories/new)**
3. Include: description, steps to reproduce, potential impact

### How to Report

1. Go to: https://github.com/file-bricks/SQLiteViewer/security/advisories/new
2. Fill out the form (title, description, severity, affected versions)
3. Submit privately (not visible to public until disclosed)

We will respond as soon as possible.

## Scope

- Database file access
- SQL execution

## Response

As a solo project, response times may vary. Critical issues will be
prioritized. Please allow reasonable time before public disclosure.
